2    * <h1>Spring Infinispan - An implementation of Spring 3.2's Cache SPI based on JBoss Infinispan.</h1>
3    * <p>
4    * Spring 3.1 introduces caching capabilities a user may comfortably utilize via a set of custom annotations, thus telling
5    * the Spring runtime which objects to cache under which circumstances.</br>
6    * Out of the box, Spring ships with <a href="">EHCache</a> as the caching provider to delegate to. It defines, however, a
7    * simple SPI vendors may implement for their own caching solution, thus enabling Spring users to swap out the default
8    * EHCache for another cache of their choosing. This SPI comprises two interfaces:
9    * <ul>
10   *   <li>
11   *     {@link org.springframework.cache.Cache <code>Cache</code>}, Spring's cache abstraction itself, and
12   *   </li>
13   *   <li>
14   *     {@link org.springframework.cache.CacheManager <code>CacheManager</code>}, a service for creating <code>Cache</code>
15   *     instances
16   *   </li>
17   * </ul>
18   * <em>Spring Infinispan</em> implements this SPI for JBoss Infinispan.
19   * </p>
20   * <p>
21   * While <em>Spring Infinispan</em> offers only one implementation of <code>org.springframework.cache.Cache</code>, namely
22   * {@link org.infinispan.spring.provider.SpringCache <code>org.infinispan.spring.provider.SpringCache</code>}, there are two implementations
23   * of <code>org.springframework.cache.CacheManager</code>:
24   * <ol>
25   *   <li>
26   *     {@link org.infinispan.spring.provider.SpringEmbeddedCacheManager <code>org.infinispan.spring.provider.SpringEmbeddedCacheManager</code>}
27   *     and
28   *   </li>
29   *   <li>
30   *     {@link org.infinispan.spring.provider.SpringRemoteCacheManager <code>org.infinispan.spring.provider.SpringRemoteCacheManager</code>}.
31   *   </li>
32   * </ol>
33   * These two implementations cover two distinct use cases:
34   * <ol>
35   *   <li>
36   *     <strong>Embedded</strong>: Embed your Spring-powered application into the same JVM running an Infinispan node, i.e. every
37   *     communication between application code and Infinispan is in-process. Infinispan supports this use case via the interface
38   *     {@link org.infinispan.manager.EmbeddedCacheManager <code>org.infinispan.manager.EmbeddedCacheManager</code>} and its default
39   *     implementation {@link org.infinispan.manager.DefaultCacheManager <code>org.infinispan.manager.DefaultCacheManager</code>}. The
40   *     latter backs {@link org.infinispan.spring.provider.SpringEmbeddedCacheManager <code>SpringEmbeddedCacheManager</code>}.
41   *   </li>
42   *   <li>
43   *     <strong>Remote</strong>: Application code accesses Infinispan nodes remotely using Infinispan's own <em>hotrod</em>
44   *     protocol. Infinispan supports this use case via {@link org.infinispan.client.hotrod.RemoteCacheManager
45   *     <code>org.infinispan.client.hotrod.RemoteCacheManager</code>}. {@link org.infinispan.spring.provider.SpringRemoteCacheManager
46   *     <code>SpringRemoteCacheManager</code>} delegates to it.
47   *   </li>
48   * </ol>
49   * </p>
50   * <strong>Usage</strong>
51   * <p>
52   * Using <em>Spring Infinispan</em> as a Spring Cache provider may be divided into two broad areas:
53   * <ol>
54   *   <li>
55   *     Telling the Spring runtime to use <em>Spring Infinispan</em> and therefore Infinispan as its caching provider.
56   *   </li>
57   *   <li>
58   *     Using Spring's caching annotations in you application code.
59   *   </li>
60   * </ol>
61   * </p>
62   * <p>
63   * <em>Register Spring Infinispan with the Spring runtime</em>
64   * <p>
65   * Suppose we want to use <em>Spring Infinispan</em> running in embedded mode as our caching provider, and suppose further that 
66   * we want to create two named cache instances, &quot;cars&quot; and &quot;planes&quot;. To that end, we put
67   * <pre>
68   * &lt;beans xmlns="http://www.springframework.org/schema/beans"
69   *        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
70   *        xmlns:cache="http://www.springframework.org/schema/cache"
71   *        xmlns:p="http://www.springframework.org/schema/p"
72   *        xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
73   *               http://www.springframework.org/schema/cache http://www.springframework.org/schema/cache/spring-cache.xsd"&gt;
74   *     &lt;cache:annotation-driven /&gt;
75   *
76   *     &lt;bean id="cacheManager" class="org.infinispan.spring.SpringEmbeddedCacheManagerFactoryBean"
77   *              p:configuration-file-location="classpath:/org/infinispan/spring/embedded/example/infinispan-sample-config.xml"/&gt;
78   *
79   * &lt;/beans&gt;
80   * </pre>
81   * in our Spring application context. It is important to note that <code>classpath:/org/infinispan/spring/embedded/example/infinispan-sample-config.xml</code>
82   * points to a configuration file in the standard Infinispan configuration format that includes sections for two named caches
83   * &quot;cars&quot; and &quot;planes&quot;. If those sections are missing the above application context will still work, yet the
84   * two caches &quot;cars&quot; and &quot;planes&quot; will be configured using the default settings defined in
85   * <code>classpath:/org/infinispan/spring/embedded/example/infinispan-sample-config.xml</code>.<br/>
86   * To further simplify our setup we may omit the reference to an Infinispan configuration file in which case the underlying
87   * {@link org.infinispan.manager.EmbeddedCacheManager <code>org.infinispan.manager.EmbeddedCacheManager</code>} will use Infinispan's
88   * default settings. 
89   * </p>
90   * <p>
91   * For more advanced ways to configure the underlying Infinispan <code>EmbeddedCacheManager</code> see 
92   * {@link org.infinispan.spring.provider.SpringEmbeddedCacheManagerFactoryBean <code>org.infinispan.spring.provider.SpringEmbeddedCacheManagerFactoryBean</code>}.
93   * </p>
94   * <p>
95   * If running Infinispan in remote mode the above configuration changes to
96   * <pre>
97   * &lt;beans xmlns="http://www.springframework.org/schema/beans"
98   *        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
99   *        xmlns:cache="http://www.springframework.org/schema/cache"
100  *        xmlns:p="http://www.springframework.org/schema/p"
101  *        xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
102  *               http://www.springframework.org/schema/cache http://www.springframework.org/schema/cache/spring-cache.xsd"&gt;
103  *     &lt;cache:annotation-driven /&gt;
104  *
105  *     &lt;bean id="cacheManager" class="org.infinispan.spring.SpringEmbeddedCacheManagerFactoryBean"
106  *              p:configuration-properties-file-location="classpath:/org/infinispan/spring/remote/example/hotrod-client-sample.properties"/&gt;
107  *
108  * &lt;/beans&gt;
109  * </pre>
110  * </p>
111  * <p>
112  * For more advanced ways to configure the underlying Infinispan <code>RemoteCacheManager</code> see 
113  * {@link org.infinispan.spring.provider.SpringRemoteCacheManagerFactoryBean <code>org.infinispan.spring.provider.SpringRemoteCacheManagerFactoryBean</code>}.
114  * </p>
115  * <em>Using Spring's caching annotations in application code</em>
116  * <p>
117  * A detailed discussion about how to use Spring's caching annotations {@link org.springframework.cache.annotation.Cacheable <code>@Cacheable</code>}
118  * and {@link org.springframework.cache.annotation.CacheEvict <code>@CacheEvict</code>} is beyond this documentation's scope. A simple example may
119  * serve as a starting point:
120  * <pre>
121  * import org.springframework.cache.annotation.CacheEvict;
122  * import org.springframework.cache.annotation.Cacheable;
123  * import org.springframework.stereotype.Repository;
124  *
125  * &#064;Repository
126  * public class CarRepository {
127  *
128  *   &#064;Cacheable("cars")
129  *   public Car getCar(Long carId){
130  *       ...
131  *   }
132  *
133  *   &#064;CacheEvict(value="cars", key="car.id")
134  *   public void saveCar(Car car){
135  *       ...
136  *   }
137  * }
138  * </pre>
139  * In both <code>&#064;Cache("cars")</code> and <code>&#064;CacheEvict(value="cars", key="car.id")</code> &quot;cars&quot; refers to the name of the cache to either
140  * store the returned <code>Car</code> instance in or to evict the saved/updated <code>Car</code> instance from. For a more detailed explanation of
141  * how to use <code>&#064;Cacheable</code> and <code>&#064;CacheEvict</code> see the relevant reference documentation 
142  * <a href="http://docs.spring.io/spring/docs/3.2.9.RELEASE/spring-framework-reference/html/cache.html">chapter</a>.
143  * </p>
144  */
